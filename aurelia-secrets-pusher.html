AURELIA · Secrets Pusher

Manage secrets, files & ZIPs across AURELIA repos

1 · Connect

Pick the target repo and paste a GitHub PAT with repo + workflow scope (classic) or Actions: read & write + Metadata: read (fine-grained). The token only lives in this browser tab unless you check “remember”.

Custom repo (override)

2 · Add or update a single secret

Encrypts the value in-browser with libsodium crypto_box_seal, then writes it via the GitHub Actions Secrets API. Existing secrets with the same name are overwritten.

Ready.

3 · Bulk push (NAME=value lines)

One secret per line as NAME=value or NAME value. Empty lines and lines starting with # are skipped. Values may be wrapped in single or double quotes; quotes are stripped.

Ready.

4 · Bulk AI key adder

Select a provider to add keys (paste one token per line) or remove existing ones. Push & Register saves each token as a numbered GitHub secret and registers it in config.json's key_registry[]. Unregister & Delete removes checked keys from both the secret store and the registry. For Cloudflare, paste account_id:token pairs — only the token is encrypted as a secret; the account ID is stored in config.json.

5 · Existing secrets in selected repo

GitHub never returns secret values — only names and timestamps. Click Use to load a name into the single-secret form for overwriting; Delete removes the secret entirely.

6 · Push ZIP to repo

Upload a .zip archive and push its contents into the repo. Each file inside the ZIP is committed individually via the Contents API. Existing files are updated (using their current SHA); new files are created. Optionally set a target subdirectory prefix. Large repos may be slow — blobs are fetched in parallel.

7 · Browse & upload files

Navigate the repo tree, view file metadata, and push individual files. Pick a file from disk and a destination path in the repo. If the path already exists it will be updated; otherwise it is created. Binary files are supported (encoded as base64 before commit).
